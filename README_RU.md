# 🌐 Список бесплатных рабочих прокси для Telegram

Обновляется автоматически, проверяется на работоспособность автоматически.

Этот репозиторий — **форк оригинального проекта** [`SoliSpirit/mtproto`](https://github.com/SoliSpirit/mtproto) с упором на удобство использования:

- сохранить оригинальный текстовый формат списка,
- и дать удобные **кликабельные Markdown и HTML списки**.

> **Источник (upstream):** `SoliSpirit/mtproto`  
> **Частота синхронизации:** каждые 12 часов

---

## 📦 Что есть в этом форке

В форке поддерживаются **три формата**:

- [`all_proxies.txt`](all_proxies.txt) — оригинальный «сырой» текст (по одному прокси на строку)
- [`all_proxies.md`](all_proxies.md) — удобный кликабельный Markdown-список со ссылками `t.me`
- [`all_proxies.html`](https://grim1313.github.io/mtproto-for-telegram/all_proxies.html) — браузерная страница с прямыми кнопками `tg://proxy`

Выбор по задаче:

- для **скриптов/автоматизации** — `all_proxies.txt`
- для **ручного использования, если `t.me` работает** — `all_proxies.md`
- для **ручного использования, если `t.me` заблокирован** — HTML-страница: [`all_proxies.html`](https://grim1313.github.io/mtproto-for-telegram/all_proxies.html)


---

## 🔄 Автообновление из оригинального репозитория

Список автоматически синхронизируется из оригинального `SoliSpirit/mtproto` каждые 12 часов.

Автоматизация реализована через:

- `scripts/sync_mtproto.py`
- `.github/workflows/sync-from-solispirit.yml`

> `all_proxies.txt` коммитится только когда его содержимое реально меняется. Если upstream не изменился, GitHub может показывать более старую дату «последнего обновления» этого файла.

## 💡 Как пользоваться

Если `t.me` открывается в вашей стране:

1. Откройте [`all_proxies.md`](all_proxies.md)
2. Нажмите на нужную ссылку прокси
3. Telegram предложит подключение

Если `t.me` заблокирован:

1. Откройте HTML-страницу: [`all_proxies.html`](https://grim1313.github.io/mtproto-for-telegram/all_proxies.html)
2. Нажмите **Open** рядом с нужным прокси, чтобы использовать прямую ссылку `tg://proxy?...`
3. Используйте **Copy**, если браузер просит вставить ссылку вручную

Markdown-список использует `https://t.me/proxy?...`, потому что GitHub и некоторые desktop-просмотрщики Markdown не всегда разрешают кликабельные нестандартные схемы вроде `tg://`.
HTML-страница предназначена для прямых ссылок `tg://proxy?...`.

Возможные нюансы:

- ссылки `tg://` требуют установленного Telegram и регистрации Telegram как обработчика таких ссылок в системе;
- некоторые браузеры или встроенные webview могут спрашивать подтверждение перед открытием Telegram или блокировать нестандартные схемы ссылок;
- если не работает ни HTML-страница, ни Markdown-список, можно вручную скопировать данные прокси из [`all_proxies.txt`](all_proxies.txt) и добавить их в настройках Telegram.
